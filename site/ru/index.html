О проекте

Rumicos — небольшое Unix-подобное ядро для x86-64, написанное с нуля на Rust (no_std, без libc). Загружается через Limine по UEFI. Это любительский исследовательский проект, а не продукт для промышленного применения.

Всё перечисленное ниже проверяется запуском настоящего ядра в QEMU, а не только модульными тестами. Текущая сборка проходит 273 хост-теста и 19 из 19 проверок реальной загрузки.

  • Процессы — copy-on-write fork(), execve() с загрузкой настоящих ELF-файлов, clone() (потоки CLONE_VM)
  • Виртуальная память — 4-уровневая трансляция, mmap/munmap, ленивые и файловые страницы, обработка copy-on-write
  • Сигналы — sigaction, kill, доставка при возврате из системных вызовов и прерываний, настоящий трамплин sigreturn
  • SMP-планировщик — загрузка нескольких ядер, многоуровневая очередь с приоритетами и work stealing, TLB-shootdown между ядрами
  • Файловая система — слой VFS с ramfs, devfs (/dev/null, zero, serial), кэш страниц, initrd в формате CPIO
  • IPC — анонимные pipe() с блокирующими чтением и записью, dup()/dup2(), ядерные futex
  • Жизненный цикл процессов — wait4(), уборка зомби, кодирование статуса выхода, SIGCHLD

Скачать

Скомпилированное ядро, готовое к загрузке. Rumicos выводит всё в последовательный порт (COM1), а не на экран — запускайте его в QEMU либо на UEFI-железе с подключённой serial-консолью.

  • rumicos-boot.zip — загрузочный пакет: UEFI-загрузчик Limine, limine.conf, ядро и инструкция INSTALL.txt (около 200 КБ)
  • kernel.elf — только ядро, для уже настроенной загрузки через Limine (или совместимый UEFI-загрузчик)

Чтобы попробовать, распакуйте архив и выполните:

qemu-system-x86_64 -machine q35 -m 512M -serial stdio -display none \
  -drive if=pflash,format=raw,readonly=on,file=OVMF_CODE.fd \
  -drive format=raw,file=fat:rw:.

Полные инструкции, включая загрузку на реальном железе, — в файле INSTALL.txt внутри архива.

Сборка из исходников

Rumicos кросс-компилируется на Linux, macOS и Windows на стабильном Rust. Ядро линкуется через rust-lld, поэтому C-тулчейн на хосте не нужен.

rustup target add x86_64-unknown-none

cargo build -p kernel --target x86_64-unknown-none --release   # собрать ядро
cargo xtask qemu-test                                          # загрузить, проверить вывод
cargo test --workspace                                         # 273 хост-теста

Исходный код, документация и инженерные заметки — в репозитории на GitHub.

Баг компилятора

Структуры в ядре незаметно портились — но только иногда и так, что ни ревью кода, ни обычные тесты этого не ловили. Трижды поле только что скопированной структуры выходило с половиной байтов, заменённых мусором.

С помощью самодельного gdbstub-клиента с аппаратными точками останова каждое повреждение было прослежено до одной последовательности инструкций: агрегатное копирование, которое rustc 1.97.0 (LLVM 22.1.6) превращал в побайтовые записи со смещением через один — записывались только чётные байты, а нечётные оставались старым мусором.

Изолированная матрица воспроизведения и сканер ассемблера указали на триггер: target-фичи +sse* в сочетании с дефолтным для этой платформы +soft-float. Исправление — однострочная правка флага сборки; сканер теперь сообщает о нуле испорченных копий во всём ядре. Это совпадает с публичной проблемой rust-lang/rust#159035. Полный разбор — в файле docs/miscompile-audit.md в репозитории.
