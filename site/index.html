About

Rumicos is a small, from-scratch Unix-like kernel for x86-64. It is written in Rust (no_std, no libc) and is booted by Limine over UEFI. It is a hobby and research project, not a production system.

Everything listed below is verified by booting the real kernel in QEMU, not only by unit tests. The current build passes 273 host tests and 19 of 19 live boot checks.

  • Processes — copy-on-write fork(), execve() loading real ELF binaries, clone() (CLONE_VM threads)
  • Virtual memory — 4-level paging, mmap/munmap, demand-zero and file-backed pages, copy-on-write faults
  • Signals — sigaction, kill, delivery on syscall and interrupt return, a real sigreturn trampoline
  • SMP scheduler — multi-core boot, multi-level feedback queue with work stealing, cross-CPU TLB shootdown
  • Filesystem — VFS layer with ramfs, devfs (/dev/null, zero, serial), a page cache, CPIO initrd
  • IPC — anonymous pipe() with blocking readers and writers, dup()/dup2(), kernel futexes
  • Process lifecycle — wait4(), zombie reaping, exit-status encoding, SIGCHLD

Download

A compiled kernel, ready to boot. Rumicos writes all of its output to the serial port (COM1), not to the screen — run it under QEMU, or on UEFI hardware with a serial console attached.

  • rumicos-boot.zip — bootable package: the Limine UEFI bootloader, limine.conf, the kernel and an INSTALL.txt guide (about 200 KB)
  • kernel.elf — just the kernel, for an existing Limine (or compatible) UEFI setup

To try it, unpack the archive and run:

qemu-system-x86_64 -machine q35 -m 512M -serial stdio -display none \
  -drive if=pflash,format=raw,readonly=on,file=OVMF_CODE.fd \
  -drive format=raw,file=fat:rw:.

Full instructions, including booting on real hardware, are in the INSTALL.txt inside the archive.

Building from source

Rumicos cross-compiles on Linux, macOS and Windows with a stable Rust toolchain. The kernel links with rust-lld, so no host C toolchain is required.

rustup target add x86_64-unknown-none

cargo build -p kernel --target x86_64-unknown-none --release   # build the kernel
cargo xtask qemu-test                                          # boot it, check serial output
cargo test --workspace                                         # 273 host tests

The source, documentation and engineering notes are in the GitHub repository.

The compiler bug

Structures in the kernel were being corrupted silently — but only sometimes, and never in a way code review or ordinary tests would catch. Three separate times a field of a freshly copied struct came out with half of its bytes replaced by garbage.

Using a custom gdbstub client with hardware watchpoints, each corruption was traced to one instruction sequence: an aggregate copy that rustc 1.97.0 (LLVM 22.1.6) lowered into byte stores at stride-2 offsets, writing only the even bytes and leaving the odd ones as stale garbage.

An isolated reproduction matrix and an assembly scanner pinned the trigger down to +sse* target features combined with this target's default +soft-float. The fix was a one-line build-flag change; the scanner now reports zero corrupted copies across the whole kernel. It matches the public issue rust-lang/rust#159035. The full write-up is in docs/miscompile-audit.md in the repository.
