About

Rumicos is an operating system for x86-64 computers. We write it from scratch in Rust — borrowing no existing kernel (Linux, BSD, or the like) and no C standard library. The kernel is the part of the system that governs the processor, memory and devices; everything else rests on it. We build it as a universal foundation, so one and the same base can serve both a desktop system and embedded devices. The first target is an operating system for ordinary personal computers.

Everything below is verified by booting the real kernel in the QEMU virtual machine, not only by unit tests in isolation. The current build passes 277 automated tests and 30 of 30 live boot checks.

What already works:

  • Processes — fork() with copy-on-write memory, running real ELF binaries via execve(), threads (clone())
  • Memory — four-level address translation, mmap/munmap, demand paging
  • Multi-core — booting on several CPU cores, a priority scheduler with cross-core load balancing
  • Filesystem — a VFS layer, an in-memory filesystem, devices (/dev), a page cache
  • Inter-process communication — pipes, signals, waiting on child processes
  • Shell — an interactive command shell with pipelines (cmd1 | cmd2), I/O redirection and program execution

Scientific and technical novelty

Rumicos is not "yet another Linux build". The difference is in the foundation:

  • Memory safety at the compiler level. Rust rules out at build time whole classes of bugs — out-of-bounds access, use-after-free, data races — that cause the majority of vulnerabilities in system software. For an operating-system kernel, where such a bug means the whole machine is compromised, this is a fundamental advantage over classic C kernels.
  • A compiler bug was found. During development, data structures began to corrupt silently — but only sometimes, and in a way neither code review nor ordinary tests caught. Using hardware watchpoint debugging, the cause was traced to a single mis-compiled instruction sequence in rustc 1.97.0. It matches the public issue rust-lang/rust#159035. That level of diagnosis speaks to the engineering depth of the project.
  • Verification on the real kernel. Every feature is confirmed by booting the actual kernel and checking its output, not only by testing functions in isolation.

Applications and market

The kernel is designed as a universal foundation for a software ecosystem. The first product on top of it is a desktop operating system: a workstation that, from the kernel up to the applications, is entirely under control and independent of foreign platforms.

Against the backdrop of migration to domestic software, demand for such solutions is forming in the public sector, in industry and in education. A kernel written in a safe language and fully open is a foundation on which application software can be built without reliance on other parties' licences or backdoors.

Roadmap

  • [done] x86-64 kernel: processes, memory, multi-core, filesystem, pipes and signals
  • [done] userspace program execution and an interactive command shell
  • [in progress] keyboard input and drivers for basic devices
  • [planned] graphical output and a windowing environment
  • [planned] networking and support for more hardware
  • [planned] a set of applications on top of the system

How the project earns. Revenue is planned not from the operating system itself but from the software that runs on it. The first target is an office suite with built-in AI from domestic developers. The kernel remains the universal foundation around which an ecosystem of Russian software is built, and that is where the business model sits.

Download and build

Rumicos cross-compiles on Linux, macOS and Windows with a stable Rust toolchain. The kernel links with rust-lld, so no separate host C toolchain is needed. For now the system writes all of its output to the serial port (COM1) — the easiest way to run it is under QEMU.

rustup target add x86_64-unknown-none

cargo build -p kernel --target x86_64-unknown-none --release   # build the kernel
cargo xtask qemu-test                                          # boot it, check the output
cargo test --workspace                                         # 277 automated tests

A ready-to-boot build and the source are in the GitHub repository.

  • rumicos-boot.zip — bootable package: the Limine UEFI bootloader, the kernel and a startup guide
  • kernel.elf — just the kernel, for an existing Limine UEFI setup
