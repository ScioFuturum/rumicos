# order relative to the other `~` lines in the same group is not checked.

# scheduler — the fork test's parent and child print concurrently, and
# both orders have been observed on known-good boots.
#
# UPDATING THIS FILE: when a feature checkpoint legitimately changes the
# boot output (e.g. init2.asm gains a new test that prints a new line),
# run `cargo xtask qemu-test`, confirm from target/qemu-test-serial.log
# that the new output is correct BY READING IT (not by pasting whatever
# appeared), then add/adjust the lines here in the same commit as the
# feature. Never delete a line just to make CI pass — a previously-green
# line going missing is exactly the regression this file exists to catch.
